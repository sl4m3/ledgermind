💡 **What**:
- Batched metadata fetching inside `MergeEngineFacade.process_batch` within `search_results` loop.
- Grouped identifiers from `search_results` and executed a single query to retrieve missing metadata, maintaining them in `full_meta_map` dictionary cache.

🎯 **Why**:
- An N+1 database I/O bottleneck existed inside the semantic search hot path loop where the engine queried `self.memory.semantic.meta.get_by_fid` iteratively for every result within the search layer. This degrades search throughput.

📊 **Impact**:
- Drastically mitigates the database round-trips over SQLite scaling N+1 database fetches to exactly 1 query inside `MergeEngineFacade.process_batch` improving search engine merge operations throughput.

🔬 **Measurement**:
- Verification run against simulated iteration benchmarking confirmed O(1) query response time.
