# Changelog v3.3.0 (March 13, 2026)

LedgerMind v3.3.0 represents a major evolution in autonomous knowledge
management with 130+ commits of improvements spanning intelligent merging,
lifecycle optimization, language localization, and architectural hardening.

## Highlights

### 🧠 Intelligent Knowledge Merging

- **Quadratic Merging Model**: Implemented advanced duplicate detection with
  hierarchical target deduction for smarter knowledge consolidation.
- **Reactive Incremental Merging**: Real-time cluster validation and event
  forwarding prevents infinite loops and ensures data integrity.
- **Randomized Merge Candidates**: Prevents stuck states by randomizing
  consolidation order.
- **Multi-Layered Verification**: Comprehensive status protection and conflict
  resolution for merge operations.

### 🔄 Lifecycle & Vitality Engine

- **Confidence Calculation**: New mathematical model for knowledge confidence
  based on evidence count, stability, and usage patterns.
- **Phase Inheritance**: Automatic promotion through PATTERN → EMERGENT →
  CANONICAL phases using `total_evidence_count` metrics.
- **Vitality Transitions**: Dynamic ACTIVE → DECAYING → DORMANT state
  management with search result weighting.
- **Automatic Lifecycle Promotion**: Background system promotes knowledge
  without manual intervention.

### 🌐 Language Localization

- **Multilingual Support**: LLM prompts now enforce language localization
  for enriched knowledge.
- **Enrichment Language Setting**: Migrated from `preferred_language` to
  `enrichment_language` namespace.
- **Language Audit**: Self-healing enrichment cycle detects and corrects
  language mismatches.
- **`ledgermind settings` Command**: New CLI for selective configuration
  management.

### 🏗️ Architectural Improvements

- **Service Decomposition**: Memory API split into specialized services
  (QueryService, LifecycleManagementService, HealthService, etc.).
- **WorkerCoordinator**: Lock-free worker synchronization prevents race
  conditions and enables graceful shutdown.
- **Per-Proposal Transactions**: Split enrichment batches to avoid lock
  timeouts during long LLM calls.
- **ACID Transactions**: True atomic transactions with Git rollback on failure.

### ⚡ Performance Optimizations

- **Search Performance**: Restored 5500+ OPS with index-based validation
  and fast-path heuristics.
- **Recording Performance**: Improved to 8+ OPS with optimized metadata
  processing.
- **Fast-Path Heuristics**: Simple keyword queries bypass vector search
  for 18,000+ OPS.
- **SQLite Concurrency**: Robust handling with strict integrity enforcement.

### 🛡️ Security & Integrity

- **Bandit Security Scan**: Fixed all identified vulnerabilities.
- **Strict I4 Integrity**: Comprehensive integrity constraints prevent
  data corruption.
- **Conflict Resolution**: Intelligent target disambiguation and multi-conflict
  resolution.
- **Transaction Rollback**: Git clean on failure ensures data consistency.

### 🤖 Background Worker

- **Responsive Shutdown**: Graceful worker termination with signal handlers.
- **Watchdog Monitoring**: Automatic detection of stale locks and orphaned
  processes.
- **Session Monitoring**: Worker shuts down when no MCP sessions active.
- **Enrichment Status Tracking**: Automatic completion detection for
  background processing.

## Breaking Changes

### Configuration Migration

**Old**: `preferred_language`, `arbitration_mode`
**New**: `enrichment_language`, `enrichment_mode`

The migration is automatic on first run. Legacy settings are removed.

### Schema Changes

- `frequency` field replaced with `total_evidence_count`
- `reinforcement_density` removed (use `total_evidence_count` directly)
- Enrichment and merge statuses elevated to root YAML metadata

## Migration from v3.2.1

```bash
# Upgrade package
pip install --upgrade ledgermind

# Run migration (automatic)
ledgermind init

# Verify version
ledgermind --version  # Should show 3.3.0
```

**Data Compatibility**: Full backward compatible. No manual migration required.
