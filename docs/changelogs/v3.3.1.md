# Changelog v3.3.1 (March 14, 2026)

LedgerMind v3.3.1 is a **critical performance and stability release** that addresses concurrency issues, optimizes database operations, and improves overall system reliability.

## 🚀 Highlights

### ⚡ Performance Optimizations

- **SQLite Index on `linked_id`** — Added database index to prevent O(N) full table scans during batch evidence link counting
- **N+1 Query Elimination** — Fixed N+1 query problem in `sync_meta_index` for 5500+ OPS search performance
- **Batch Query Optimization** — Improved EpisodicStore batch operations with pre-fetched link counts
- **Search Performance Restoration** — Restored 5500+ OPS search and 8+ OPS recording throughput

### 🔒 Security & Concurrency

- **TOCTOU Race Condition Fix** — Addressed time-of-check-time-of-use vulnerability in concurrent writes
- **SQLite DB Locked Errors** — Resolved database locking issues in concurrent environments
- **Global Thread Lock Registry** — Implemented path-based synchronization for FileSystemLock
- **Manual Write Blocking** — Added protection against manual writes when background worker is active

### 🛠️ Bug Fixes

- **list_active_conflicts** — Now correctly returns only 'active' status records (not 'draft')
- **reconcile_untracked** — Fixed to sync meta index after recovering untracked files
- **_update_meta_for_file** — Fixed dead code and missing MemoryLoader.parse import
- **sync_meta_index** — Now always checks hash mismatches, not just file differences
- **HttpError in CI** — Fixed PR security comment handling

### 🎨 UX Improvements

- **CLI Error Surfacing** — ledgermind-mcp CLI errors are now visible to users
- **Test Stability** — Improved concurrency test reliability with retries and absolute paths
