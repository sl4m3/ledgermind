# Changelog v3.3.6 (April 12, 2026)

LedgerMind v3.3.6 delivers major performance improvements, enhanced security, and a polished CLI experience with rich visual feedback.

## 🚀 Highlights

### ⚡ Massive Performance Optimization

- **N+1 Query Elimination** — Resolved critical N+1 query bottlenecks across multiple subsystems including enrichment facade, search queries, metadata fetching, and event processing.
- **SQLite Batch Optimization** — Implemented chunked batch operations and optimized SQLite queries using `json_each` for faster database operations.
- **Memoization** — Added memoization for lifecycle weight calculations in search queries to reduce redundant computations.
- **Deduplication** — Optimized batch database operations by deduplicating input IDs, preventing unnecessary duplicate queries.

### 🛡️ Critical Security Fixes

- **Subprocess Security** — Fixed multiple insecure subprocess execution vulnerabilities across CRITICAL, HIGH, and MEDIUM severity levels (#108, #109, #110).
- **Authentication Hardening** — Added missing authentication to direct MCP tools (#118) and admin endpoints (#106).
- **Path Traversal Protection** — Fixed path traversal vulnerability in `purge_memory` (#123) and path hijacking in git subprocess calls using `shutil.which` (#121).

### 🎨 Enhanced CLI Experience

- **Rich Console Formatting** — Standardized CLI output formatting with `rich` library for better readability and visual consistency (#127, #129).
- **Command Palette** — Added "show output" command to command palette for easier access to processing results (#119).
- **Status Bar Improvements** — Made status bar clickable to show logs (#113) and added visual error states for better debugging (#116).
- **Output Channel** — Replaced intrusive error popups with output channel for cleaner UX (#95).
