# Changelog v3.3.4 (March 16, 2026)

LedgerMind v3.3.4 is a significant update focused on multi-client isolation, security hardening, and performance optimization of memory resolution.

## 🚀 Highlights

### 🛡️ Namespace Isolation (Multi-Client Support)

- **Client Isolation** — Introduced the `--cli` flag across all bridge commands (`bridge-context`, `bridge-sync`, `bridge-record`). This allows multiple AI clients (Claude, Gemini, Cursor) to share the same physical storage while maintaining logical separation of their interaction history and context.
- **Smart Hook Installation** — All installers (`ClaudeInstaller`, `GeminiInstaller`, `CursorInstaller`, `VSCodeInstaller`) now automatically configure hooks to use the correct client namespace.
- **Namespace-Aware Background Worker** — The background maintenance process now tracks which client initiated it, enabling better resource management and logging in multi-agent environments.

### 🔒 Security Hardening (Project Sentinel)

- **Command Injection Protection** — Fixed potential subprocess command injection risks in the health check and installer modules.
- **Input Validation** — Tightened validation for file paths and shell arguments across the server component.

### ⚡ Performance Optimization (Project Bolt)

- **Fast ID Resolution** — Optimized hierarchical ID resolution in the query service, resulting in faster context retrieval for deeply nested memory structures.
- **Efficient Background Processing** — Reduced overhead of background metadata synchronization and integrity checks.

## 🔧 Technical Details

### Namespace Implementation in Bridge API

The `IntegrationBridge` now accepts an optional `namespace` parameter that filters search results and tags new memories:

```python
# Internal routing logic
results = self._memory.search_decisions(
    prompt, 
    limit=limit, 
    mode="balanced", 
    namespace=self.namespace  # Isolation boundary
)
```

### Security Patch for Subprocesses

Replaced risky shell executions with structured argument lists in `installers.py`:

```python
# Before (vulnerable)
# subprocess.run(f"claude mcp add ... --path {path}", shell=True)

# After (secure)
subprocess.run(
    ["claude", "mcp", "add", "ledgermind", "ledgermind-mcp", "--", "--path", os.path.abspath(path), "--client", client],
    capture_output=True, text=True
)
```

```bash

```
