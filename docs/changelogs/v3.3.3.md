# Changelog v3.3.3 (March 15, 2026)

LedgerMind v3.3.3 is a maintenance release that introduces an interactive settings manager, improves OpenRouter integration, and enhances JSON parsing robustness for LLM outputs.

## 🚀 Highlights

### 🛠️ Interactive Settings Manager

- **New `ledgermind settings` UI** — Added a full-featured interactive mode for managing configuration. You can now view, change, and reset settings through a user-friendly terminal interface.
- **Default Action** — Running `ledgermind settings` without subcommands now automatically enters interactive mode.
- **Improved UX** — Real-time validation for settings like `merge_threshold` and clear descriptions for each configuration key.

### 🌐 Enhanced OpenRouter Integration

- **Reliable Cloud Enrichment** — Fixed a critical `AttributeError` in the `OpenRouterClient` that prevented cloud-based enrichment from functioning correctly.
- **Model Flexibility** — Improved support for various models through the OpenRouter API, ensuring consistent behavior across different LLM providers.

### 🧠 Robust Knowledge Processing

- **Resilient JSON Parsing** — Upgraded the `ResponseParser` with advanced "healing" logic. It can now automatically fix common LLM formatting errors such as unescaped quotes and literal newlines within JSON strings.
- **Safe Fallbacks** — The new parsing logic only triggers when standard JSON loading fails, ensuring no impact on valid responses from high-quality models.

### Advanced JSON Healing

The `ResponseParser` now applies targeted regex-based fixes to malformed JSON payloads:

```python
# Example of quote healing for LLM strings
fixed = re.sub(r'(?<=[а-яА-Яa-zA-Z0-9])"(?=[а-яА-Яa-zA-Z0-9\s])', r'\\"', text)
```
