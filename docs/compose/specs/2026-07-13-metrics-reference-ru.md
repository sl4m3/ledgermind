## Обзор

Этот документ описывает все метрики, используемые в пайплайне жизненного цикла LedgerMind. Каждая метрика описана с:
- **Назначение**: Что измеряет эта метрика
- **Расчёт**: Как вычисляется
- **Диапазон**: Допустимые значения
- **Триггеры обновления**: Когда эта метрика изменяется
- **Влияние**: На какие решения влияет эта метрика

## [M1] Уверенность (Confidence) (0.0 - 1.0)

### Назначение
Как часто это предложение использовалось (инжектилось в контекст). Чем выше — тем надёжнее.

### Расчёт
```python
уверенность = min(1.0, log1p(количество_использований) / 2.3)
```

### Шкалы
- 0 использований → 0.0
- 1 использование → 0.30
- 10 использований → 1.0 (максимум)
- 100 использований → 1.0 (максимум)

### Логика
Больше инъекций = надёжнее. Насыщается на 10 использованиях.

### Триггеры обновления
- Записано использование (поиск/inject)

### Влияние
- **Скорость decay**: Низкая уверенность → быстрый decay
- **Gate merge**: Уверенность < 0.5 → пропустить merge
- **Proposal → Decision**: Уверенность > 0.7 обязательна
- **Переходы vitality**: Уверенность < 0.9 → DECAYING

## [M2] Балл стабильности (Stability Score) (0.0 - 1.0)

### Назначение
Насколько стабильны доказательства во времени. Чем выше — тем стабильнее.

### Расчёт
```python
if количество_доказательств < 2:
    стабильность = 0.0

elif количество_доказательств >= 2:
    интервалы = [дни между последовательными подтверждениями]
    дисперсия = variance(интервалы)
    дельта_стабильность = max(0.0, 1.0 - (дисперсия / (время_жизни + 1.0)))
    фактор_возраста = min(1.0, время_жизни_дни / 7.0)
    стабильность = дельта_стабильность × (0.5 + 0.5 × фактор_возраста)
```

### Интерпретация
- Высокая стабильность: Доказательства поступают регулярно
- Низкая стабильность: Доказательства поступают нерегулярно
- Фактор возраста: Новые решения не могут иметь высокую стабильность (нужно 7+ дней)

### Триггеры обновления
- Тик пайплайна (пересчёт)
- Добавлено новое доказательство (интервал изменяется)

### Влияние
- **Уверенность**: 40% веса
- **Продвижение фазы**: Стабильность >= 0.7 обязательна для CANONICAL
- **Gate merge**: Стабильность < 0.3 → пропустить merge

## [M3] Количество доказательств (Total Evidence Count)

### Назначение
Количество слитых proposals (рекурсивный подсчёт).

### Расчёт
```python
def count_evidence(proposal):
    """Рекурсивно считает все слияния."""
    count = len(proposal.supersedes)
    for fid in proposal.supersedes:
        merged = get_proposal(fid)
        if merged:
            count += count_evidence(merged)
    return count
```

### Логика
Больше слияний = больше подтверждений этого знания.

### Триггеры обновления
- Proposal сливается с другим proposal

### Влияние
- **Продвижение фазы**: evidence >= 50 для EMERGENT, >= 150 для CANONICAL
- **Proposal → Decision**: evidence >= 50 обязательна
- **Gate merge**: evidence < 5 → пропустить merge

## [M4] Количество использований (Hit Count)

### Назначение
Сколько раз это решение использовалось (искалось, инжектилось в контекст).

### Расчёт
```python
количество_использований += 1  # Каждый раз когда решение извлекается/используется
```

### Триггеры обновления
- Векторный поиск вернул это решение
- Решение инжектилось в контекст LLM
- Ручное извлечение

### Влияние
- **Уверенность**: Компонент использования (20% веса)
- **Полезность**: бонус_использования = min(0.4, log1p(количество_использований) / 10.0)

## [M5] Последнее использование (Last Hit At)

### Назначение
Временная метка последнего использования.

### Расчёт
```python
последнее_использование = datetime.now()  # Каждый раз когда решение используется
```

### Триггеры обновления
- Векторный поиск вернул это решение
- Решение инжектилось в контекст LLM

### Влияние
- **Vitality**: последнее_использование > 30 дней → DECAYING
- **Переактивация vitality**: последнее_использование < 7 дней → ACTIVE
- **Полезность**: штраф_свежести если последнее_использование > 7 дней

## [M6] Покрытие (Coverage) (0.0 - 1.0)

### Назначение
Какую часть окна наблюдения это решение наблюдалось.

### Расчёт
```python
покрытие = время_жизни_дни / окно_наблюдения_дни  (по умолчанию: 30 дней)
```

### Интерпретация
- 0.0: Только создано (0 дней)
- 0.5: Наблюдалось 15 дней
- 1.0: Наблюдалось 30+ дней

### Триггеры обновления
- Тик пайплайна (пересчёт)

### Влияние
- **Продвижение фазы**: покрытие >= 0.2 для EMERGENT, >= 0.3 для CANONICAL

---

## [M7] Оценочная полезность (Estimated Utility) (0.0 - 1.0)

### Назначение
Насколько полезно это знание предположительно.

### Расчёт
```python
полезность = стабильность × 0.3 + уверенность × 0.5 + покрытие × 0.2
```

### Триггеры обновления
- Тик пайплайна (пересчёт)
- Записано использование
- Добавлено доказательство

### Влияние
- **Инжекция контекста**: Высокая полезность → высокий ранг
- **Ранжирование поиска**: Высокая полезность → высокая релевантность

## [M8] Фаза (PATTERN → EMERGENT → CANONICAL)

### Назначение
Уровень зрелости знания.

### Значения
- **PATTERN**: Сырой гипотеза, только создана
- **EMERGENT**: Проверена, есть доказательства
- **CANONICAL**: Полностью зрелая, авторитетная

### Правила продвижения

#### PATTERN → EMERGENT
```python
# Стандартный путь
if доказательства >= 50 AND покрытие >= 0.2:
    продвинуть()

# Альтернативный путь
if уверенность >= 0.5 AND доказательства >= 30:
    продвинуть()
```

#### EMERGENT → CANONICAL
```python
# Стандартный путь
if доказательства >= 150 AND стабильность >= 0.7 AND покрытие >= 0.3:
    продвинуть()

# Альтернативный путь
if уверенность >= 0.75 AND стабильность >= 0.7:
    продвинуть()
```

### Триггеры обновления
- Тик пайплайна (проверка продвижения)
- Добавлено доказательство

### Влияние
- **Gate merge**: Та же фаза обязательна для merge
- **Инжекция контекста**: Высокая фаза → высокий приоритет

---

## [M9] Жизнеспособность (Vitality) (ACTIVE → DECAYING → DORMANT)

### Назначение
Актуально ли это знание ещё.

### Значения
- **ACTIVE**: Актуально, использовалось недавно
- **DECAYING**: Теряет актуальность, не использовалось давно
- **DORMANT**: Может быть устаревшим, очень низкая уверенность

### Правила переходов

#### ACTIVE → DECAYING
```python
if уверенность < 0.9 OR последнее_использование > 30 дней:
    жизнеспособность = DECAYING
```

#### DECAYING → ACTIVE (реактивация)
```python
if последнее_использование < 7 дней:
    жизнеспособность = ACTIVE
```

#### DECAYING → DORMANT
```python
if уверенность < 0.2:
    жизнеспособность = DORMANT
```

#### DORMANT → УДАЛЕНИЕ
```python
if уверенность < 0.1 AND дни_в_состоянии_dormant > 7:
    удалить()
```

### Триггеры обновления
- Тик пайплайна (проверка жизнеспособности)
- Записано использование
- Изменение уверенности

### Влияние
- **Скорость decay**: DORMANT → быстрый decay
- **Gate merge**: DORMANT → пропустить merge
- **Удаление**: DORMANT + низкая уверенность → удалить

## [M10] Время жизни (Lifetime Days)

### Назначение
Как давно существует это знание.

### Расчёт
```python
время_жизни_дни = (последнее_видение - первое_видение).total_seconds() / 86400
```

### Триггеры обновления
- Тик пайплайна (пересчёт)

### Влияние
- **Покрытие**: покрытие = время_жизни_дни / 30
- **Стабильность**: фактор_возраста = min(1.0, время_жизни_дни / 7.0)

---

## [M11] Первое/Последнее видение (First Seen / Last Seen)

### Назначение
Временные границы этого знания.

### Расчёт
```python
первое_видение = min(даты_подтверждений)
последнее_видение = max(даты_подтверждений)
```

### Триггеры обновления
- Создание: первое_видение = сейчас, последнее_видение = сейчас
- Новое доказательство: первое_видение/последнее_видение обновляются
- Использование: последнее_видение обновляется

### Влияние
- **Время жизни**: время_жизни_дни = последнее_видение - первое_видение
- **Стабильность**: интервалы между подтверждениями

## Зависимости метрик

| Метрика | Зависит от | Используется в |
|---------|------------|----------------|
| Уверенность | Количество использований | Decay, Merge, Vitality, Decision |
| Стабильность | Интервалы доказательств, Время жизни | Фаза, Merge |
| Количество доказательств | Рекурсивный подсчёт слияний | Фаза, Proposal → Decision, Merge |
| Количество использований | Использование | Уверенность, Полезность |
| Последнее использование | Использование | Vitality, Полезность |
| Покрытие | Время жизни | Фаза |
| Полезность | Стабильность, Доказательства, Использования, Последнее использование | Ранжирование поиска, Инжекция контекста |
| Фаза | Доказательства, Стабильность, Покрытие | — |
| Vitality | Уверенность, Последнее использование | Decay, Merge, Удаление |

## Сводка порогов для решений

| Переход | Обязательные метрики |
|---------|----------------------|
| Proposal → Decision | уверенность > 0.7 AND доказательства >= 50 |
| PATTERN → EMERGENT | доказательства >= 50 OR (уверенность >= 0.5 AND доказательства >= 30) |
| EMERGENT → CANONICAL | доказательства >= 150 AND стабильность >= 0.7 AND покрытие >= 0.3 |
| ACTIVE → DECAYING | уверенность < 0.5 OR последнее_использование > 30 дней |
| DECAYING → DORMANT | уверенность < 0.2 |
| DORMANT → УДАЛЕНИЕ | уверенность < 0.1 AND дни_в_состоянии_dormant > 7 |
| Пропустить Merge | уверенность < 0.5 OR стабильность < 0.3 OR доказательства < 5 |
| Минимальное удержание | доказательства < 5 AND дни_с_создания < 14 |
